1 Getting Startedtop

1.1 Install your Plugin


Once you have downloaded the Vallejo Menu package, there are 2 ways of installing this plugin. These are as follows:

METHOD 1
  1. Unzip the download package using 7-zip or equivalent.
  2. Using a FTP client (eg. FileZilla), upload its contents to your Wordpress Installation. You will need to place the folder into the following location: wp-content/plugins
  3. Now inside your Wordpress admin area, go to Plugins and there you will see the Vallejo Menu plugin.
  4. Click Activate to activate the plugin. If you do not see the plugin in this screen, you have done something incorrectly. Review the last few steps again.
METHOD 2
  1. The Vallejo download package is a zip file named vallejo-menu-plugin.zip, this is a compressed version of the plugin.
  2. Inside your Wordpress admin area, go to Plugins > Add New > Upload Plugin
  3. Click Browse, locate the vallejo-menu-plugin.zip file and upload it
  4. Now you will see the Vallejo Menu plugin in the Plugins list.
  5. Click Activate to activate the plugin. If you do not see the plugin in this screen, you have done something incorrectly. Review the last few steps again.

Congratulations you have now successfully installed the Vallejo Menu Wordpress Plugin

2 Plugin Structure - How It Workstop

The Vallejo Menu plugin is essentially adding two menu entries to the Wordpress dashboard:
  1. Menu Items: add Menu Items and Menu Categories.
  2. Settings > Vallejo Menu Plugin: configure your Menus and the setup the Auto Share functionality.


3 Create a Menutop

The Vallejo Menu plugin features a Menu Item custom post type. This can be found in the main Wordpress admin menu under Menu Items. To display menu items in the Menu page, you must first create menu items here.


3.1 Menu Item


To create a Menu Item do the following:
  1. Select the Menu Items item in the main admin menu & select "Add New".
  2. Give the Menu Item a title.
  3. Enter a description of the menu item into the Wordpress editor. The description will only be displayed on the Menu Item page. The Menu in itself will not show the description, use the short description field instead (see next step). A link to each Menu Item page is provided in the Menu page though.
  4. In the next box entitled "Menu item info" you have 4 fields to add information to your menu item:
    Short Description Short descriptions are displayed in the menu.
    Price Item price (without currency).
    Icon 1 Select an icon from the list.
    Label 1 Enter the icon label.
    The "Icon 2" & "Label 2" fields will only appear once you have selected the first icon and saved your changes.
  5. In the Menu Category box on the left part of the screen, give your menu item a category. Enter the first letters of an existing category to select it or create a new one directly from here.
  6. You may use the Featured Image box to add a photo of the menu item's. It will be displayed next to the menu item in the menu.
  7. Now click Publish to publish this menu item. This type of post is mostly used to provide content for the Menu page. The post may also be viewed in it's own page however by clicking the View Post button.
You can manage your menu categories in the Menu Items > Menu Categories page. Each menu item must belong to a category in order to appear in the Menu.

3.2 Setup a Menu


To setup a Menu do the following:
  1. Go to Settings > Vallejo Menu Plugin
  2. Select the Menu Category you would like to appear first from the Menu Category 1 dropdown list.
  3. Click the Save Settings button.
  4. A new Menu Category 2 field will then appear for you to add Menu Categories to that same menu.
Menu Items inside a Category are ordered by Date.

3.3 Add a Menu page


To add a Menu page do the following:
  1. Go to Settings > Vallejo Menu Plugin
  2. Copy the shortcode of the desired menu, for example: [vmenu number=1].
  3. Add a new page (go to Pages > Add New) and paste the [vmenu number=1] shortcode in the main textarea field.
  4. Now click Publish to publish the page.

4 Vallejo Menu Plugin Settingstop

In this section we will explain the custom settings that come with the Vallejo Menu plugin, where to find them and what you would use them for.

Vallejo Menu Plugin's custom settings can be found at Settings > Vallejo Menu Plugin. These settings will appear once the Vallejo Menu plugin has been successfully installed. The various settings are organized into tabs. Simply click on a tab to show its contents in the main content area.
In the admin panel itself, beside each option, you can see an information tooltip containing a short description of what that particular option does. In most cases these short descriptions will be enough to explain how to use the custom settings found in this plugin. Nonetheless all settings are explained here below.

4.1 Menus Settings


Menus settings will let you configure and add Menus to your website. This tab is open by default when the page is loaded.

Currency This option allows you set the currency symbol you would like to appear before the Menu Items prices.
Display Columns This option allows you to display Menu Items distributed in 1, 2 or 3 columns.
Menu Category X This field allows the selection of a Menu Category. The order in which the categories are displayed in the Menu page will be determined by the number of the category field. After saving the first category, a new Menu Category field will appear, and below it, the Menu 2 section.

4.2 Auto Share Settings


This tab controls the Auto Share to social media platforms feature.

Activate Check the box to Activate the Auto Share feature.
Once activated, the Auto Share tab will display settings to setup and manage sharing over the following social media platforms: Facebook & Twitter. You will find inline help and directions to activate social sharing for your Facebook and Twitter accounts. Once correctly setup, a new box named Auto Share will appear on the Post / Page / Menu Item edit screen as shown in the screenshot below.



4.3 Auto Share Logs


This tab only appears once the Auto Share is activated. It will display the last five logs of each social media account.

Post ID The post ID (can belong to a page, a menu item or a post).
Account type Your configured Facebook or Twitter account.
Published On Publication date and time.
Status If a network or authentication error occured during the sharing process, the error message will be displayed here.
The table rows in red indicate an error occured during the sharing process.



5 Plugin Translationtop

The only method to translate this plugin is using a POT file.

A POT file is a simple method but it is designed to basically translate the plugin's static text content into another language. For example this would be used when the plugin was designed to be used in English, but your website will be in Italian or another language, and you would like to translate all the parts of the plugin which are not the content you will be writing. This would included back end translation such as admin field names, descriptions etc, as well as front end static text which is visible to the visitor. These front end items would includ things like contact form field names, comment section titles, and other titles which appear automatically rather then being actually entered by you. This method is not designed to handle language switching in your website. If you require this more advanced functionality, you will need to use a plugin. No specific plugins have been tested with this plugin, and hence we cannot guarantee that they will work.


5.1 Translation using POT File


To translate the plugin using the POT file do the following:

  1. Create a .po file for your language, for example de_DE.po
  2. Copy all text from provided plugin .pot file into your .po file
  3. Edit the .po file with Poedit http://www.poedit.net/
  4. Create the .mo file
  5. Edit the wp-config.php file in your installation and set WPLANG so it matches your languages: define('WPLANG', 'de_DE');
  6. The Plugin should now display all messages in your language.
BACKUP YOUR LANGUAGE PO/MO FILES Upon plugin update, language files are NOT preserved. You will need to manually restore your po/mo files right after the update.
